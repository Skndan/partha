# Integration: GitHub

Partha will treat GitHub as the source of truth for implementation detail while keeping planning inside Partha.

## Goals

- Auto-create repositories when projects graduate intake stages.
- Link commits and PRs to issues/milestones for traceability.
- Emit changelog drafts based on merged history + MCP-driven metadata ([../automations/auto-changelog.md](/docs/automations/auto-changelog)).

## Draft architecture notes

- GitHub App vs OAuth App trade-offs for multi-workspace installs.
- Webhooks for `push`, `pull_request`, `release`.
- Mapping GitHub org/repo ↔ Partha workspace/project keys without collisions.

## Dependencies

- Workspace secrets vault per integration (planned).
- Background worker tier for webhook ingestion (planned).
