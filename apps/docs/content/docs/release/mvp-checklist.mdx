Use this checklist before promoting Partha to production. Automated gates run in GitHub Actions ([`.github/workflows/ci.yml`](../../.github/workflows/ci.yml)).

## A. Preflight (automated)

Run from the repository root:

```bash
bun install
docker compose up postgres -d
export DATABASE_URL=postgresql://postgres:postgres@localhost:5432/partha
export BETTER_AUTH_SECRET=local-dev-secret-local-dev-secret-local-12
export BETTER_AUTH_URL=http://localhost:4000
export NEXT_PUBLIC_URL=http://localhost:4000
export SKIP_ENV_VALIDATION=1
export RUN_INTEGRATION_TESTS=1

cd apps/app && bun run db:push && cd ../..
bun run typecheck
bun run lint
cd apps/app && bun run test && bun run test:integration
cd apps/app && bun run db:validate && bun run knip
bun run build:app && bun run build:site
```

| Step | Command | Pass |
|------|---------|------|
| Typecheck | `bun run typecheck` | ☐ |
| Lint | `bun run lint` | ☐ |
| Unit tests | `cd apps/app && bun run test` | ☐ |
| Integration tests | `cd apps/app && bun run test:integration` | ☐ |
| DB validate | `cd apps/app && bun run db:validate` | ☐ |
| Knip | `cd apps/app && bun run knip` | ☐ |
| Build app + site | `bun run build:app && bun run build:site` | ☐ |

## Known MVP limitations (documented, not blockers)

- No MCP **resources** registered (tools only).
- API keys and session revocation APIs exist without settings UI.
- Notifications API exists; dedicated notifications UI may be limited.
- Platform admin / election master data routes are **not** part of this MVP.
