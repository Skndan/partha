Register Partha as an HTTP MCP endpoint (`/api/mcp`) alongside other servers; config file locations vary by Claude Desktop release and OS. Complete OAuth when the client opens a browser for login—same pattern as Cursor’s hosted MCP.

## Pattern

```json
{
  "mcpServers": {
    "partha": {
      "url": "https://your-domain.com/api/mcp"
    }
  }
}
```

Consult Anthropic’s latest Claude Desktop documentation for the precise config file location and schema on your OS.

## Authentication

Use Partha’s OAuth flow ([oauth.md](/docs/mcp/oauth)). After Claude triggers an MCP call, complete login/consent in the browser window Claude opens.

## Notes

- Keep scopes narrow (`mcp:read`, `workspace:read`, etc.) until automation requires writes.
- Workspace-bound tokens simplify multi-tenant setups.
